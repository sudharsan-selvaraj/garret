# Garret project instructions

## UI / design (always apply)

Build UI that looks and feels like a native macOS app.

- **Minimal.** Show what the user needs. Nothing decorative.
- **Copy.** Short labels. No filler sentences, no em-dash padding, no marketing tone. A control's purpose should be obvious from a 1 to 3 word label; add a hint only when genuinely needed.
- **Layout.** Keep windows readable and uncluttered. Generous spacing, clear hierarchy, one primary action per view.
- **Group, don't dump.** Segregate items into logical groups. Never put everything in one window or one long list. Use sections, sidebars, tabs, or separate panes so each screen has one job.
- **Unify only related things.** Merge flows or components only when they are genuinely the same concern. Do not force unrelated items together for the sake of unification.

Apply this to every settings pane, dialog, manager, and widget surface. When a screen starts to feel crowded, split it by logical group before adding more.
