Part 1 ГӮВ· Product

Garret ГўВҖВ” the programmable desktop layer for macOS

Garret turns the empty space behind your windows into glanceable widgets, summons them all above everything with a keystroke, and lets anyone build a widget for it ГўВҖВ” from a weather card to a full device controller.

ГӮВ·The problem

The info you glance at all day ГўВҖВ” calendar, PRs, builds, clipboard, a connected phone ГўВҖВ” lives in a dozen apps, tabs, and menu-bar drips. The context-switch tax is constant, and the most valuable pixels on screen (the desktop) do nothing but hold a wallpaper. And if the widget you need doesn't exist, you're stuck ГўВҖВ” you can't just build it.

ГӮВ·The three modes

One layer, surfaced at the level of attention you want.

Г°ВҹВҢВӨГҜВёВҸ

Ambient

Widgets live on the desktop layer ГўВҖВ” above the wallpaper, behind your apps. Always there, never in the way.

calendar ГӮВ· PRs ГӮВ· CI ГӮВ· system load
ГўВҡВЎ

HUD

Hit ГўВҢВҳГўВҮВ§Space and the whole layer floats above every app ГўВҖВ” even a full-screen Space ГўВҖВ” ready to act, then dismiss.

check & act without leaving your app
ГўВҢВҳ

Action palettes

Focused, keystroke-summoned tools that do a thing and get out of the way.

clipboard ГӮВ· device control ГӮВ· command runner

ГӮВ·The ecosystem ГўВҖВ” Garret is a container

Garret ships the layer, the HUD, and the plumbing, then gets out of the way. Anyone builds a widget and shares it as a .garret file. Two tiers, one authoring model, and the tier is chosen by what a widget needs ГўВҖВ” you don't pick it.

web ГӮВ· sandboxed

Isolated, strict policy, gets exactly the access it declares (a network host, an account, storage) and nothing more. One-click trust.

native ГӮВ· full access

Raw Node in an isolated process ГўВҖВ” run programs, touch files, talk to USB. Off by default; runs only after a plain "full access, no sandbox" confirmation.

Part 2 ГӮВ· Building an extension (proposed @garretapp/sdk)

Authoring guide

One SDK, one project shape, one UI programming model ГўВҖВ” for both a safe web widget and a full-access native extension. You declare capabilities; Garret derives the tier and the consent.

01The model

An extension is always a UI (a webview you write) and an optional Host (a Node backend, only when you need system access). They talk over one small, typed bridge ГўВҖВ” request/response, events, streams. The only difference between a web widget and a native extension is whether you ship a host/. Everything the UI can do has the same API either way.

UI (webview, no Node) ГўВҮВ„ bridge ГўВҮВ„ Host (your raw-Node backend, full access) ГўВҶВҗ native UI (webview, no Node) ГўВҮВ„ bridge ГўВҮВ„ Platform (Garret-brokered capabilities) ГўВҶВҗ web
You declareГўВҖВҰTierConsent
UI only, or safe caps (storage, service:*, network:<host>)web / sandboxedDisclosure at install
a host/ and a system cap (process/fs/native/network:*)full accessDefault-OFF, typed confirm
Tier rule (resolved ГўВҖВ” ARCH-06 / Q3): require BOTH The host entry is the runtime signal (spawn a child?); explicit capabilities are the consent signal (what to tell the user). Full-access needs both. A host with no system cap is rejected at install ("host requires a capability"); a system cap with no host is rejected too ("process requires a host"). network:* also requires a host.

06Host API ГўВҖВ” @garretapp/sdk/host

defineHost hides the entire bridge envelope + sends ready only after your (possibly async) factory resolves. Cross-reference sibling methods with function declarations ГўВҖВ” they hoist, so no extra machinery:

host/index.ts
import { defineHost } from '@garretapp/sdk/host'
import { readdir } from 'node:fs/promises'

export default defineHost<Api, Events>((ctx) => {
  // function declarations hoist ГўВҶВ’ safe to cross-reference (the P9 fix ГўВҖВ” no `methods` arg)
  async function list({ dir }: { dir: string }) { return { items: await readdir(dir) } }
  function run({ argv }: { argv: string[] }) {
    return ctx.stream((out, signal) => {
      const child = ctx.spawn(argv, { signal })   // array only, env-scrubbed, auto-tracked
      child.stdout.on('data', d => out.push(d.toString()))
      child.on('close', code => out.end({ code }))
    })
  }
  async function restart(a: { argv: string[] }) { await stop(); return run(a) } // plain call
  function stop() { /* ГўВҖВҰ */ }
  return { list, run, restart, stop }
})

08aStreaming

Declare it in the contract by returning Stream<Chunk, Result> (Result defaults to void); produce with ctx.stream; consume with a StreamCall (or useStream).

shared/api.ts
import type { Stream } from '@garretapp/sdk'
export interface Api {
  list(a: { dir: string }): { items: string[] }
  run(a: { argv: string[] }): Stream<string, { code: number }>   // Chunk, Result
}
export interface Events { changed: { dir: string } }
const run = host.run({ argv })   // StreamCall ГўВҖВ” NOT a promise (can't await an endless stream)
run.onData(c => append(c)).onEnd(({ code }) => done(code)).onError(showErr)
// run.cancel()  ГӮВ·  await run.result()  (bounded streams only)
Guarantees (from the review + critic passes) Stream start returns synchronously (can't be killed by the request timeout) ГӮВ· pre-.onData chunks are bounded-buffered + a stream nobody consumes auto-cancels ГӮВ· StreamCall is deliberately not thenable ГӮВ· backpressure coalesces; filter firehoses host-side.

10Lifecycle & instances

  • One host per placed instance. Removal/disable ГўВҶВ’ SIGTERM ГўВҶВ’ ctx.onDispose ГўВҶВ’ spawned children auto-killed ГўВҶВ’ SIGKILL after 3s.
  • config is per-instance; storage/secrets are per-extension (shared across placements) ГўВҖВ” atomic + key-level merged. Same-key writes from two placements still clobber ГўВҶВ’ use g.instanceStorage for per-placement state (SDK-06).
  • Payloads are structured-clone (Date/Map/Uint8Array survive; functions/class instances/handles don't). Binary travels as Uint8Array.
  • Persistence lives outside the integrity-checked code dir; removed on uninstall.
  • Power: when the board is ambient (not HUD), polling is throttled and (via the SDK active signal) widgets should pause rAF/animations.

13Full example ГўВҖВ” Device Control

shared/api.ts
import type { Stream } from '@garretapp/sdk'
export interface Device { serial: string; model: string }
export interface Api {
  devices(): { devices: Device[] }
  mirror(a: { serial: string }): Stream<string, { exit: number }>
}
export interface Events { deviceAdded: Device; deviceRemoved: { serial: string } }
host/index.ts
import { defineHost } from '@garretapp/sdk/host'
export default defineHost<Api, Events>((ctx) => {
  const track = ctx.spawn(['adb', 'track-devices'])
  track.stdout.on('data', () => { /* diff ГўВҶВ’ ctx.emit('deviceAdded' | 'deviceRemoved', ГўВҖВҰ) */ })
  ctx.onDispose(() => track.kill())
  async function devices() {
    const adb = await ctx.resolveBinary('adb')   // BINARY_NOT_FOUND w/ per-OS hint
    return { devices: parse(await ctx.spawn([adb, 'devices', '-l']).text()) }
  }
  function mirror({ serial }: { serial: string }) {
    return ctx.stream((out, signal) => {
      const p = ctx.spawn(['scrcpy', '-s', serial], { signal })
      p.stderr.on('data', d => out.push(d.toString()))
      p.on('close', code => out.end({ exit: code }))
    })
  }
  return { devices, mirror }
})

Architecture & decisions

1 ГӮВ· Window level model ГўВҖВ” ГўВңВ… solved (harder than it looks)

One BrowserWindow in two states: Ambient (desktop level, below apps, above wallpaper, click-through) and HUD (above everything incl. full-screen Spaces, focusable). Pure setAlwaysOnTop(true,'desktop') was not enough ГўВҖВ” it didn't survive full-screen Spaces and showed a Mission Control tile. We use a native addon (native/mac_window.mm ГўВҶВ’ garret_mac.node, wrapped by macWindow.ts):

  • pinToDesktop ГўВҶВ’ window.level = kCGDesktopIconWindowLevel+offset, collectionBehavior = CanJoinAllSpaces | Stationary | IgnoresCycle.
  • raiseToHud ГўВҶВ’ kCGScreenSaverWindowLevel + CanJoinAllSpaces | FullScreenAuxiliary | Stationary.
  • makePanel ГўВҶВ’ a non-activating NSPanel ГўВҖВ” the one window kind macOS floats over another app's full-screen Space without activating / Space-switching.
  • createWindow.ts: setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true }) ГўВҖВ” the critical flag. Ambient click-through: setIgnoreMouseEvents(true, { forward: true }); window transparent.
Subtle trap: level authority Electron's setAlwaysOnTop and the native window.level must agree or they fight. Fix: the addon is the single authority, and the desktop re-pin on show is guarded while HUD is up (index.ts setHud). Keep geometry stable across the level switch ГўВҖВ” the dim/rise feel is CSS, not a resize. Residual nuance (ARCH-05): HUD sits at screen-saver level (above Notification/Control Center) ГўВҖВ” test floating if that ever looks wrong.

2 ГӮВ· IPC bridge protocol ГўВҖВ” Г°ВҹВҹВЎ envelope done, streaming wire not frozen

Two hops: UI (renderer) ГўВҮВ„ main ГўВҮВ„ native host (utilityProcess); web widgets do only the left hop. The SDK is a thin wrapper over this wire, so it must be right before SDK code. Right on the fundamentals today (extensionHost.ts): {t:'req'} ГўВҶВ’ {t:'res'} + {t:'event'}; structured clone (never manual JSON ГўВҖВ” Uint8Array/Date/Map survive); per-instance correlation (one host per placement, own seq); synchronous stream registration.

Open (adopt before SDK): first-class stream_* wire frames instead of layering over the event channel, and bridge-level backpressure (don't push 10k adb logcat chunks). Target envelope:

{ t:'req', id, method, args }   { t:'res', id, result }   { t:'err', id, code, message, hint? }
{ t:'stream_start', id, method, args }   { t:'chunk', id, data }   { t:'stream_end', id, result }
{ t:'stream_err', id, code, message }    { t:'cancel', id }        { t:'event', channel, payload }
// ids namespaced ${instanceId}:${ulid()}

3 ГӮВ· Renderer isolation ГўВҖВ” ГўВҡВ ГҜВёВҸ the one open foundational decision

Each widget UI needs its own isolated renderer (process, session, CSP); the Electron primitive here is the hardest thing to change later. We're all-in on <webview> (NativeWidget.tsx, SandboxWidget.tsx). Isolation is strong for the web tier ГўВҖВ” sandbox/session.ts uses a per-widget partition with setPermissionRequestHandler(deny-all) + onBeforeRequest cancelling non-garret-widget: requests, all in main. (Native shares one persist:garret-native partition ГўВҖВ” fine, it's full-access/trusted.)

<webview> (current)WebContentsView (proposed)
IsolationGood (own process/session/CSP)Best; officially supported
Board integrationA DOM element ГўВҖВ” position/drag/z-order/scroll free from the gridManual geometry ГўВҖВ” hand-sync bounds every frame
Electron directionDiscouraged, historically buggyThe future
DECISION OPEN Stay on <webview> (keep layout simplicity, accept the tradeoff) or migrate to WebContentsView now (better isolation, take on geometry-sync). The security gap is smaller than it looks (enforcement is main-side regardless); the real cost is geometry management for a movable grid. A spike measured that cost: happy path small, cost concentrates in scroll + DPI/multi-monitor. Decide before the SDK freezes the render contract.

4 ГӮВ· Native host lifecycle ГўВҖВ” Г°ВҹВҹВЎ core done, two gaps

Target: fork ГўВҶВ’ host {t:'ready'} ГўВҶВ’ 10s ready-timeout ГўВҶВ’ steady state ГўВҶВ’ remove/disable ГўВҶВ’ SIGTERM ГўВҶВ’ ctx.onDispose ГўВҶВ’ 3s ГўВҶВ’ SIGKILL. Gaps (build in SDK step-4) Graceful shutdown ladder ГўВҖВ” today kill() is an immediate child.kill(); need SIGTERM ГўВҶВ’ onDispose ГўВҶВ’ 3s ГўВҶВ’ SIGKILL. Env scrub ГўВҖВ” delete GARRET_* before spawn and from ctx.spawn children (matters once the SDK injects a secret-key env var).

5 ГӮВ· Capability enforcement ГўВҖВ” ГўВңВ… solved (already the intended model)

declared in manifest ГўВҶВ’ verified at install ГўВҶВ’ enforced in main, never the renderer. sandbox/net.ts (SSRF/private-IP/allowlist) + session onBeforeRequest + permission-deny. The install record is the authoritative ceiling ГўВҖВ” host-written, HMAC-signed, never the user-writable manifest. Consent is a signed record; any capability/code change re-triggers it (native: sha delta ГўВҶВ’ enabled:false + re-consent; sandbox: added-permissions re-prompt). Full-access hosts get unrestricted ctx.fetch ГўВҖВ” the consented bargain. Remaining: scrub GARRET_* (Pillar 4).

6 ГӮВ· Power / visibility ГўВҖВ” Г°ВҹВҹВЎ throttle built; occlusion signal pending

Garret is an always-on desktop layer; macOS treats it as "visible" even when covered, so OS throttling may not kick in ГўВҖВ” the real battery risk, not the render primitive (measured 0.0% CPU at rest; the primitive is power-neutral). Built: a HUD-gated poll throttle (scheduler.ts setBoardActive stretches every job's interval ГғВ—4 when ambient, snaps to full rate + refresh on HUD; first poll still immediate); wired via setHud ГўВҶВ’ setBoardActive + focus/blur; backgroundThrottling: true explicit; full pause on system sleep.

Gaps A true occlusion signal (HUD/focus is a heuristic proxy). The SDK active signal (useGarret().active) so widget UIs pause rAF/animations ГўВҖВ” build with the SDK. Verify under load with powermetrics. Tunable: IDLE_MULTIPLIER.
